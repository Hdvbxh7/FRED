    EvaluateurJUnit.java

    Package

    LibEvaluateur.EvaluationsUnitaire

    Description

    Concrete implementation of EvaluateurUnitaire that executes JUnit 4 tests and collects detailed results including pass/fail status, error messages, and execution time.

    Purpose

    Automatically runs JUnit tests on student code and provides comprehensive feedback on test results, helping identify which tests passed and why others failed.

    Inheritance

    Evaluateur → EvaluateurUnitaire → EvaluateurJUnit

    Fields

    fichiersTests

    List of test files to execute. Can be:

    • .class files (pre-compiled test classes)
    • .java files (source files - will look for corresponding .class)

    classpaths

    Additional paths for class loading:

    • JUnit library JARs
    • Student code directories
    • Test dependencies

    Constructors

    EvaluateurJUnit(ArrayList<File> fichiers)

    Creates evaluator with test files only.

    Parameters:

    • fichiers - List of test files (.java or .class)

    Classpath: Uses default classpath (current directory and parent)

    EvaluateurJUnit(ArrayList<File> fichiers, ArrayList<String> classpaths)

    Creates evaluator with test files and custom classpath.

    Parameters:

    • fichiers - List of test files
    • classpaths - Additional paths for class loading

    Use Case: When tests depend on external libraries or specific build directories

    Key Methods

    evaluer()

    Executes all JUnit tests and captures comprehensive results.

    Process:

    1. Prepare ClassLoader

      • Add test file directories to URLs
      • Add explicit classpaths
      • For .java files, check for ../bin directory
      • Create URLClassLoader with all URLs

    2. Redirect Output

      • Capture System.out and System.err
      • Allows capturing test print statements

    3. Execute Tests

      • For each test file:
        • Extract class name from filename
        • Load class dynamically
        • Run with JUnitCore
        • Collect results
        • Record pass/fail status

    4. Restore Output

      • Return System.out/err to normal

    5. Format Results

      • Store detailed results in resultat
      • Store pass/fail array in testsResultat

    Error Handling:

    • ClassNotFoundException: Logs and marks file as failed
    • General exceptions: Logs and marks as failed

    ResultatVersTAP()

    Intended to convert results to TAP format.

    Status: 🚧 TODO - Not yet implemented

    Comment in code: "TODO: Conversion vers format TAP (JP c'est ta tête sur l'affiche)"

    obtenirNomClasse(File fichier)

    Private method to extract class name from file.

    Parameters:

    • fichier - Test file (.java or .class)

    Returns: Class name without extension

    Examples:

    • StudentTest.java → StudentTest
    • MathTest.class → MathTest

    JUnit Execution Flow

    1. Test files → [StudentTest.class, MathTest.class]
2. Create EvaluateurJUnit(files, classpaths)
3. Call evaluer()
   ├─ Build URLClassLoader
   │  ├─ Add test directories
   │  ├─ Add ../bin if exists
   │  └─ Add custom classpaths
   ├─ Redirect System.out/err
   ├─ For each test file:
   │  ├─ Extract class name
   │  ├─ Load class dynamically
   │  ├─ Create JUnitCore instance
   │  ├─ Run tests: junit.run(testClass)
   │  ├─ Collect Result object
   │  │  ├─ getRunCount() - total tests
   │  │  ├─ getFailureCount() - failed tests
   │  │  ├─ wasSuccessful() - all passed?
   │  │  └─ getFailures() - failure details
   │  └─ Add to testsResults
   ├─ Restore System.out/err
   └─ Store formatted results
4. Results available via getResultat() / getTestsResultat()

    Result Format

    resultat String Structure:

    === Exécution des tests de StudentTest ===
Tests exécutés: 10
Tests réussis: 8
Tests échoués: 2
Temps d'exécution: 156 ms

--- Échecs ---
Test: testDivisionByZero
Message: Expected exception: java.lang.ArithmeticException
Trace: [stack trace]

Test: testNullPointer
Message: null
Trace: [stack trace]

=== Exécution des tests de MathTest ===
...

=== Sortie des tests ===
[captured System.out/err output]

    testsResultat Array:

    JUnit Result Object

    Key Methods Used:

    • result.getRunCount() - Total number of tests executed
    • result.getFailureCount() - Number of failed tests
    • result.getRunTime() - Execution time in milliseconds
    • result.wasSuccessful() - True if all tests passed
    • result.getFailures() - List of Failure objects

    Failure Object:

    • getTestHeader() - Test name
    • getMessage() - Failure message
    • getTrace() - Full stack trace

    ClassLoader Configuration

    URL Sources:

    1. Test File Parent Directory

    2. Bin Directory (if exists)

    3. Custom Classpaths

    Why Dynamic Loading?

    • Tests may be in different packages
    • Avoids hardcoding class references
    • Supports flexible directory structures

    Output Capture

    Purpose: Capture any System.out.println() or System.err.println() in tests.

    Error Scenarios

    Class Not Found:

    ERREUR: Classe non trouvée - StudentTest
Message: StudentTest

    → File not in classpath or wrong package

    Execution Error:

    ERREUR lors de l'exécution de StudentTest: NullPointerException

    → Test initialization failed

    Critical Error:

    ERREUR CRITIQUE: Could not create URLClassLoader

    → Classpath misconfiguration
