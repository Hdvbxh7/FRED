    EvaluateurCompilationAda.java

    Package

    LibEvaluateur.EvaluationsCompilation

    Description

    Concrete implementation of EvaluateurCompilation for compiling Ada source files using the GNAT compiler (gnatmake).

    Purpose

    Verifies that student Ada code compiles successfully using the Ada 2012 standard with all warnings enabled.

    Inheritance

    Evaluateur → EvaluateurCompilation → EvaluateurCompilationAda

    Fields

    fichiers

    • Index 0: Ada source file to compile (.adb or .ads)
    • Remaining: Additional Ada files or dependencies

    Constructors

    EvaluateurCompilationAda(File binaire)

    Creates evaluator for a single Ada file without additional dependencies.

    Parameters:

    • binaire - The Ada source file to compile (e.g., program.adb)

    EvaluateurCompilationAda(File binaire, ArrayList<File> dependences)

    Creates evaluator with additional Ada source files or dependencies.

    Parameters:

    • binaire - Main Ada source file
    • dependences - List of additional Ada files required for compilation

    Key Methods

    evaluer()

    Executes Ada compilation using gnatmake and processes results.

    Process:

    1. Create StringWriter to capture compiler output
    2. Call compileAda() with the source file path
    3. Close writer and get output string
    4. Pass output to resultatVersTAP()

    Throws: Exception for I/O or compilation errors

    resultatVersTAP(String SortieTest)

    Converts GNAT compiler output to TAP format.

    Logic:

    • Success: Output is EMPTY (no messages)
      • Sets testsResultat[0] = true
      • resultat = "Test réussi"
    • Failure: Output contains ANY text (errors or warnings)
      • Sets testsResultat[0] = false
      • resultat = "Test raté"

    Important: Even warnings cause failure, ensuring strict code quality.

    Parameters:

    • SortieTest - Raw output from gnatmake

    Debug: Prints output with delimiters: Voici la sortie +++++ [output] +++++

    compileAda(BufferedWriter out, String file)

    Private static method that executes the GNAT compiler.

    Parameters:

    • out - Writer to capture compiler output
    • file - Absolute path to Ada source file

    Compilation Command:

    Options:

    • -D [directory] - Output directory for compiled files
    • -gnat2012 - Use Ada 2012 language standard
    • -Wall - Enable all warnings

    Process:

    1. Creates output directory if it doesn’t exist
    2. Builds ProcessBuilder with gnatmake command
    3. Redirects stderr to stdout
    4. Reads all output from process
    5. Waits for compilation to complete
    6. If exit code ≠ 0, writes error output to buffer
    7. If exit code = 0 (success), buffer remains empty

    Error Handling:

    • Catches exceptions and writes error message to output
    • Non-zero exit codes indicate compilation failure

    Compilation Flow

    1. Student writes Ada code → chiffre_significatif.adb
2. Create EvaluateurCompilationAda(file)
3. Call evaluer()
   ├─ Call compileAda()
   │  ├─ Create output directory
   │  ├─ Build gnatmake command
   │  ├─ Execute process
   │  ├─ Capture output
   │  └─ Check exit code
   └─ Call resultatVersTAP()
      ├─ Check if output is empty
      ├─ Set testsResultat[0]
      └─ Set resultat string
4. Results available via getResultat() / getTestsResultat()

    Success vs Failure

    Successful Compilation:

    (no output from gnatmake - exit code 0)

    → Empty string → testsResultat[0] = true

    Compilation with Errors:

    chiffre_significatif.adb:15:03: error: "Put_Line" is not visible
chiffre_significatif.adb:15:03: error: non-visible declaration at a-textio.ads:263

    → Non-empty output → testsResultat[0] = false

    Compilation with Warnings:

    chiffre_significatif.adb:20:10: warning: variable "Count" is not referenced

    → Non-empty output → testsResultat[0] = false

    Note: Warnings are treated as failures to enforce code quality standards.

    Output Directory

    Compiled artifacts (.ali, .o, executables) are written to:

    LibEvaluateur/EvaluationsCompilation/Compiled_Code/

    Directory is automatically created if it doesn’t exist.

    Limitations

    1. Error Detail Loss: resultat is just “Test réussi” or “Test raté”

      • Actual compiler messages only appear in debug output
      • Consider capturing full output in resultat for student feedback

    2. Warnings as Failures: Strict but might frustrate students

      • Consider separate handling for warnings vs errors

    3. Single Test Result: Only one boolean (compiled / didn’t compile)

      • Could be extended to count error types
